# WORKLOG

이 문서는 작업 진행 중 메모를 수집한다.  
`CHANGELOG.md`에는 최종 동작 변화만 기록하고, 중간 시도/가설/실패 원인은 이 문서에 남긴다.

## 기록 규칙
- 날짜/요청 단위로 기록한다.
- 사실(관측)과 추론(가설)을 분리해 적는다.
- 실패한 시도도 남기되, 원인과 다음 액션을 함께 적는다.
- 민감정보(토큰/비밀번호/개인정보)는 기록하지 않는다.

## 예시

### [2026-02-27] 요청: Windows 에디터 목록 노출 불안정 확인
- 담당: codex
- 관련 브랜치/커밋: `main` / `-`
- 관련 파일:
  - `src/main/main.js`
  - `src/renderer/renderer.js`

#### 1) 목표
- 설치된 에디터 목록이 빈 목록으로 보이지 않도록 안정화.

#### 2) 관측 사실
- 분할 전환 직후 드롭다운에 `설치된 에디터 없음`이 간헐적으로 표시됨.

#### 3) 가설
- IPC 조회 응답 지연 시 렌더러가 빈 상태를 캐시함.

#### 4) 시도 내역
1. 시도: 렌더러에 pending query 재사용 로직 추가.
   - 결과: 동일 시점 중복 조회는 감소했으나 초기 빈 상태는 여전히 발생.
2. 시도: fallback 목록 즉시 렌더 후 조회 완료 시 교체.
   - 결과: 초기 빈 상태 재현 불가.

#### 5) 검증
- 실행 명령:
  - `node --check src/main/main.js`
  - `node --check src/renderer/renderer.js`
- 결과:
  - 모두 통과, 수동 재현 절차에서 빈 목록 증상 미재현.

#### 6) 결론
- 초기 렌더 fallback + 비동기 교체 방식으로 증상 완화 가능.

#### 7) 잔여 리스크 / 다음 액션
- 실제 사용자 환경(저성능/백신 개입)에서 추가 검증 필요.
